# 后台变更同步

> 本目录用于处理“后台原型已修改，需要反向同步已有 PRD”的场景。原则是先生成差异报告，再按确认结果更新 PRD，不直接覆盖已有文档。

## 适用场景

- 后台原型页面新增、删除或调整后，需要同步到 PRD。
- 原型中的字段、按钮、筛选、表格、弹窗、抽屉、状态或权限发生变化。
- 已有 PRD 需要与最新后台原型重新对齐。

## 目录结构

```text
变更同步/系统名称/
└── 后台管理端/
    ├── 变更差异报告.md
    └── 同步记录.md
```

## 同步规则

- 不直接覆盖已有 PRD，必须先输出差异报告。
- 未能从原型确认的字段、规则、权限、状态，不写成确定结论，标记为待确认。
- 文案、样式、布局微调只在影响用户理解或业务规则时同步到 PRD。
- 字段、按钮、入口、状态、校验、权限、流程变化必须同步到 PRD。

## 推荐提示词

```text
使用 html-prototype-to-prd，对比 原型/后台管理端/系统名称/ 和 PRD/系统名称/后台管理端/，生成变更同步差异报告，输出到 变更同步/系统名称/后台管理端/变更差异报告.md。先不要修改 PRD。
```

```text
根据 变更同步/系统名称/后台管理端/变更差异报告.md 中我确认的同步项，更新 PRD/系统名称/后台管理端/ 和 PRD/系统名称/00-功能模块架构.md，并输出同步记录到 变更同步/系统名称/后台管理端/同步记录.md。
```
