# 后台管理端项目模板

这是一套只面向后台管理系统的项目启动模板，适合运营后台、管理控制台、业务审核台、配置后台等 PC 管理端项目。

## 技术栈

| 类型 | 约定 |
|---|---|
| 前端框架 | Vue 3 + Vite |
| UI 规范 | Ant Design Vue |
| 工程入口 | `src/main.js`、`src/App.vue` |
| 原型目录 | `原型/后台管理端/系统名称/` |
| PRD 目录 | `PRD/系统名称/后台管理端/` |

## 本地开发

本项目已包含真实 `Vue 3 + Vite + Ant Design Vue` 工程骨架。首次运行前先安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

构建生产版本：

```bash
npm run build
```

## 使用方式

1. 复制整个 `后台管理端模板` 文件夹到新项目位置。
2. 将目录名 `系统名称` 替换成真实系统名。
3. 在 `原始需求/需求草稿.md` 写入业务背景、目标和初始想法。
4. 在 `设计规范/00-全局规范.md` 和 `设计规范/01-后台管理端规范.md` 补充品牌、组件、布局和验收规则。
5. 按 `PRD/系统名称/后台管理端/M01-模块名称/F001-功能名称.md` 的格式生成 PRD。
6. 根据 PRD 生成后台 HTML 原型，输出到 `原型/后台管理端/系统名称/`。

## 推荐提示词

```text
基于 原始需求/需求草稿.md、设计规范/00-全局规范.md 和 设计规范/01-后台管理端规范.md，按 PRD/系统名称/后台管理端/M01-模块名称/F001-功能名称.md 的结构访谈我并写一份后台管理端 PRD，输出到 PRD/系统名称/后台管理端/
```

```text
使用 prototype-generator｜原型生成，根据 PRD/系统名称/后台管理端、设计规范/00-全局规范.md 和 设计规范/01-后台管理端规范.md 生成后台管理端 HTML 原型，输出到 原型/后台管理端/系统名称/。技术风格必须遵守 Vue 3 + Ant Design Vue，不得使用 Vant 风格组件。
```

```text
使用 diagram-generator｜图表生成，根据 PRD/系统名称/00-功能模块架构.md 和 PRD/系统名称/后台管理端/，生成后台管理端系统功能架构图、核心业务流程图和关键功能操作流程图，输出到 交付物/图表/系统名称/。
```

```text
使用 html-prototype-to-prd｜原型反推，对比 原型/后台管理端/系统名称/ 和 PRD/系统名称/后台管理端/，生成变更同步差异报告，输出到 变更同步/系统名称/后台管理端/变更差异报告.md。先不要修改 PRD。
```

## 目录约定

- `原始需求/`：最初业务想法、访谈记录、用户输入。
- `VERSION`：当前项目版本号。
- `CHANGELOG.md`：每次需求、设计、PRD、原型和交付物修改的迭代记录。
- `package.json`：前端工程依赖和脚本。
- `package-lock.json`：前端依赖锁定文件。
- `vite.config.js`：Vite 构建配置。
- `src/`：真实 Vue 3 前端工程源码。
- `设计规范/00-全局规范.md`：品牌、色彩、字体、状态、文案等共用规范。
- `设计规范/01-后台管理端规范.md`：后台管理端布局、组件、交互和验收规则。
- `PRD/系统名称/00-功能模块架构.md`：后台功能基线。
- `PRD/系统名称/后台管理端/`：按模块和功能拆分的后台 PRD。
- `原型/后台管理端/系统名称/`：后台管理端 HTML/CSS/JS 静态原型。
- `变更同步/系统名称/后台管理端/`：原型变更反向同步 PRD 的差异报告和同步记录。
- `交付物/图表/`：流程图、架构图、时序图和数据模型图。
- `交付物/操作手册/后台管理端/`：后台管理员操作手册。
- `.codex/skills/`：模板内置 skill。

## 版本管理

每次有效迭代都需要同时维护：

1. 更新 `VERSION` 中的当前版本号。
2. 在 `CHANGELOG.md` 追加本次修改记录。
3. 提交 Git 时使用能对应迭代记录的提交说明。

记录内容应包含：

- 修改日期。
- 版本号。
- 新增、调整、修复或移除的内容。
- 影响到的 PRD、设计规范、原型、交付物或配置文件。

版本号建议使用 `主版本.次版本.修订号`。新增模块、页面、PRD、原型或交付物时递增次版本；小范围文案、样式或规则修正时递增修订号。

## 防混用规则

- 只使用 Ant Design Vue 风格组件。
- 不使用 Vant、底部 TabBar、移动端 Picker、移动端 Popup。
- 不把移动端页面说明写入后台 PRD。
- 原型生成前必须明确目标目录是 `原型/后台管理端/系统名称/`。
