# 03-演示素材规范

> 本文件定义 HTML 原型中的演示图片、头像、Banner、上传预览、票据附件等 mock 素材规则。目标是让原型更接近高保真业务场景，避免灰色占位块和随机无关图片。

## 使用优先级

| 优先级 | 方式 | 适用场景 |
|---:|---|---|
| 1 | 在线公共素材库 | 多项目复用、在线演示、团队协作 |
| 2 | 本地 mock 图片素材包 | 离线演示、客户现场评审 |
| 3 | 主题图片 URL | 快速生成业务化原型 |
| 4 | 随机占位图 URL | 早期草稿、非关键图片 |
| 5 | 灰色占位块 | 仅限骨架态或明确无图状态 |

正式交付原型优先使用在线公共素材库；需要离线演示时，再同步成本地 mock 图片副本。不要每个项目重复手工维护一套图片。

## 推荐复用方式

推荐维护一个可在线访问的公共素材库，同时保留本地源文件。例如：

```text
~/MockAssets/
├── products/
├── avatars/
├── banners/
├── receipts/
├── stores/
├── uploads/
├── empty/
└── icons/
```

线上部署后的访问地址示例：

```text
https://assets.example.com/mock-assets/
```

原型中可直接引用：

```html
<img src="https://assets.example.com/mock-assets/products/product-001.jpg" alt="商品图">
```

项目需要离线演示时，再同步到当前项目：

```bash
工具/同步mock素材.sh ~/MockAssets backend
```

这样素材只维护一份，多项目在线复用；需要离线时再复制一份到原型目录。

## 在线素材库配置

在线素材库说明和 manifest 示例见：

```text
在线素材库/README.md
在线素材库/manifest.example.json
原型/素材库配置.example.js
```

推荐在原型入口页面加载配置：

```html
<script src="./素材库配置.example.js"></script>
```

然后根据 `window.MOCK_ASSET_CONFIG.mode` 选择在线或本地地址。

## 本地素材目录

本地素材目录是同步后的项目内副本，不建议作为长期唯一素材源。

后台管理端：

```text
原型/后台管理端/系统名称/assets/images/mock/
├── products/
├── avatars/
├── banners/
├── receipts/
├── stores/
├── uploads/
├── empty/
└── icons/
```

## 分类与尺寸

| 英文目录 | 中文含义 | 推荐尺寸 | 适合放什么 |
|---|---|---|---|
| `products/` | 商品图片 | 100x100、160x160、750x750 | 商品缩略图、商品主图、商品列表图 |
| `avatars/` | 头像图片 | 80x80、120x120 | 用户头像、员工头像、客户头像 |
| `banners/` | 横幅图片 | 1440x320、1200x260 | 后台工作台 Banner、运营图、顶部宣传图 |
| `receipts/` | 单据票据 | 320x200、640x400 | 票据、合同、附件、发票、单据预览 |
| `stores/` | 门店现场 | 343x180、640x360 | 门店、仓库、现场照片、设备场景图 |
| `uploads/` | 上传预览 | 160x120、320x240 | 上传组件预览图、图片附件、凭证照片 |
| `empty/` | 空状态图 | 240x160、320x200 | 无数据、无搜索结果、暂无内容插画 |
| `icons/` | 业务图标 | 64x64、128x128 | 自定义业务图标、功能入口图标 |

## 目录中文备注

```text
products/   商品图片：商品图、物料图、产品主图
avatars/    头像图片：用户、员工、客户头像
banners/    横幅图片：首页 Banner、活动图、运营图
receipts/   单据票据：合同、发票、票据、附件预览
stores/     门店现场：门店、仓库、设备、现场照片
uploads/    上传预览：上传组件里的图片、凭证、附件
empty/      空状态图：暂无数据、无搜索结果、空列表
icons/      业务图标：功能入口、自定义业务小图标
```

## 命名规则

文件名使用英文、数字和短横线，不使用中文和空格。

```text
products/product-001.jpg           商品图片 001
products/product-002.jpg           商品图片 002
avatars/avatar-001.png             头像图片 001
banners/banner-001.jpg             后台横幅 001
receipts/receipt-001.jpg           单据票据 001
stores/store-001.jpg               门店现场 001
uploads/upload-001.jpg             上传预览 001
empty/empty-list.png               空列表状态图
```

## 引用规则

后台管理端原型：

```html
<img src="./assets/images/mock/products/product-001.jpg" alt="商品图">
```

CSS 建议：

```css
.mock-image {
  width: 100%;
  height: 100%;
  object-fit: cover;
  display: block;
}
```

## 高保真要求

- 图片比例必须和容器比例一致或接近，避免拉伸变形。
- 列表中同类图片保持统一尺寸。
- 商品图、门店图、票据图不要混用。
- 头像使用头像类素材，不使用风景、商品、票据图片。
- Banner 需要保留文字安全区，不让文字贴边或被按钮遮挡。
- 上传预览框必须固定尺寸，避免图片加载后页面跳动。
- 图片缺失时使用 `empty/` 中的空状态图或清晰占位态。

## 外链备用规则

当本地素材不足时，可临时使用外链：

```text
商品/业务图：https://loremflickr.com/343/180/product?random=101
门店/仓库图：https://loremflickr.com/343/180/store?random=102
头像：https://api.dicebear.com/9.x/initials/svg?seed=张三
普通占位：https://picsum.photos/343/180?random=101
```

外链图片只用于临时原型。正式演示前建议替换为本地 mock 图片。
